Latar Belakang dan Permasalahan

Pusat data konvensional memiliki keterbatasan dalam hal skalabilitas, ketersediaan layanan, dan efisiensi pengelolaan sumber daya. Pengelolaan infrastruktur fisik secara mandiri memerlukan biaya tinggi serta kompleksitas operasional yang besar.

Cloud computing hadir sebagai solusi dengan menyediakan sumber daya komputasi berbasis layanan. Proyek ini mensimulasikan penerapan cloud computing melalui web application berbasis Platform as a Service (PaaS) yang berjalan sepenuhnya pada lingkungan cloud.

Arsitektur Cloud

Alur Arsitektur Sistem:

User → Internet → CDN + SSL → Cloud PaaS (GitHub Pages) → Client-side Data Processing → Visualization Dashboard

Arsitektur ini menunjukkan penerapan resource abstraction, scalability, availability, dan fault tolerance yang merupakan karakteristik utama pusat data cloud modern.

Proses pengolahan data dilakukan sepenuhnya di sisi klien (client-side), sehingga aplikasi ini berfungsi sebagai Web Application sekaligus Data Processing System berbasis cloud.

Konfigurasi Keamanan

  • HTTPS / SSL untuk enkripsi komunikasi data
  • Isolasi platform pada lingkungan cloud PaaS
  • Tidak ada backend server yang terekspos
  • Penerapan shared responsibility model
