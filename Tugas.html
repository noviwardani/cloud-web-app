Masalah Konvensional (On-Premise)

Pengelolaan pusat data fisik membutuhkan investasi CAPEX (Capital Expenditure) tinggi untuk hardware, pendinginan, dan listrik. Skalabilitas terbatas pada fisik server yang tersedia, dan risiko downtime tinggi jika terjadi kegagalan hardware tunggal (Single Point of Failure).

Solusi Cloud Computing

Mengubah model ke OPEX (Operational Expenditure). Menggunakan resource yang diabstraksi (Virtualization) memungkinkan Elasticity (naik/turun otomatis) dan High Availability. Proyek ini mendemonstrasikan model PaaS di mana infrastruktur diurus oleh provider (GitHub).

Catatan Arsitektur Data Center:
  • Scalability: GitHub Pages menggunakan CDN (Fastly) yang mendistribusikan konten ke berbagai Edge Server di seluruh dunia.
  • Fault Tolerance: Jika satu server down, traffic dialihkan ke node lain dalam jaringan CDN.
  • Resource Abstraction: Kita tidak perlu memikirkan OS server, patching, atau kabel jaringan. Fokus hanya pada aplikasi/data.

Implementasi Keamanan Cloud

SSL/TLS Encryption

Seluruh komunikasi data antara user dan server dienkripsi menggunakan protokol HTTPS. Mencegah serangan Man-in-the-Middle.

Platform Isolation

Aplikasi berjalan di lingkungan Sandbox browser (Client-side), tidak memiliki akses langsung ke backend database, meminimalisir risiko injeksi SQL.

Shared Responsibility

Provider (GitHub): Mengamankan fisik, jaringan, dan OS.
User (Mahasiswa): Mengamankan kode, aset data, dan konfigurasi repositori.
